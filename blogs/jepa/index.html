从 I-JEPA 到 V-JEPA 2.1：
预测式视觉表征学习的四步演进

JEPA（Joint-Embedding Predictive Architecture，联合嵌入预测架构）是 Yann LeCun 提出的"通往自主机器智能"蓝图中的核心组件。它的主张只有一句话： 不要在像素空间重建，也不要靠人造数据增广，而要在抽象表征空间中"预测"——用一部分输入的表征去预测另一部分输入的表征。 这一原理从图像（I-JEPA）出发，扩展到视频（V-JEPA），再被规模化为可理解、预测、规划的世界模型（V-JEPA 2），最终解决稠密特征质量问题（V-JEPA 2.1）。

1 · JEPA 核心原理：在隐空间里"预测"

要理解整个系列，先要分清自监督视觉学习的三条技术路线。JEPA 是对前两条的"折中再超越"：

路线代表怎么学痛点
生成式 / 重建MAE, BEiT, VideoMAE, data2vec在像素 / token 空间重建被遮挡的内容被迫拟合不可预测的低层细节，特征语义性低、收敛慢
不变性 / 对比DINO, iBOT, MSN, SimCLR强制对人造增广视图保持表征不变依赖手工增广（注入偏置、跨模态难迁移）、需多视图、需负样本
JEPA（本系列）I/V-JEPA用 context 的表征预测 target 的表征（隐空间回归）无需增广、无需负样本、无需像素重建；语义更强、更省算力

1.1 三个网络 + 一个停梯度 = 不坍缩的预测器

所有四篇论文共用同一套"骨架"，差异只在模态、掩码维度、损失加在哪些 token、规模：

图 1 · JEPA 通用计算图：context-encoder 可学（接收梯度），target-encoder 是其 EMA 副本且对输出 stop-gradient，predictor 在隐空间回归 target 表征。非对称结构 + EMA + 停梯度共同防止表征坍缩。

1.2 目标函数：从 L2 到 L1，再到"全 token"

# I-JEPA：M 个 target 块上的平均 L2（patch 级）
L = (1/M) · Σi=1..M Σj∈Bi ‖ ŝyj − syj ‖22
# V-JEPA / V-JEPA 2：被掩码位置上的 L1（更稳定）
minimizeθ,φ ‖ Pφ( Eθ(x), Δy ) − sg( EMAθ(y) ) ‖1
# V-JEPA 2.1：对"所有" token 监督（掩码 M + 可见 C），距离加权
Lpredict = (1/|M|) Σi∈M ‖ predi − sg(targeti) ‖1
Lcontext = (1/|C|) Σi∈C λi ‖ predi − sg(targeti) ‖1,   λi = λ / √( dmin(i, M) )
Ldense = Lpredict + Lcontext
为什么不坍缩？target-encoder 不是凭空固定的——它是 context-encoder 权重的指数滑动平均（EMA / Polyak），再对其输出施加 stop-gradient。 这种"师生 + 停梯度 + 预测器"的非对称结构（BYOL 式机制）阻止了"所有表征都塌成同一常数"的平凡解。I-JEPA 明确指出 EMA target-encoder 是用 ViT 训练 JEPA 的必要条件；后续工作沿用了固定 EMA 系数 0.99925。

1.3 实证：隐空间预测 vs 像素预测 / 增广不变

vs 像素重建（MAE 类）

I-JEPA 的关键消融极具说服力（ViT-L/16，IN-1% 评测）：

  • 预测 target-encoder 输出表征 → 66.9%（500ep）
  • 预测像素 → 40.7%（800ep）
  • 掩码加在 target-encoder 输出 vs 输入：67.3 vs 56.1（ViT-H/16）

隐空间预测可丢弃不可预测的低层像素细节，特征更语义、收敛快 ~5×。

vs 增广不变（DINO/iBOT 类）

JEPA 不用任何手工增广、不用负样本、单视图即可：

  • I-JEPA ViT-H/16448 线性探测 81.1%，超过靠增广的 iBOT ViT-L/16（81.0）
  • 训练 ViT-H/14 比 ViT-S/16 的 iBOT 还快 >2.5×
  • V-JEPA 在 SSv2 运动理解上比 DINOv2 ViT-g/14 高 +21 分（增广不变模型结构上学不到运动）

2.1 动机

当时两条主流 SSL 路线各有硬伤（见 §1 表）。I-JEPA 要做的是把 LeCun 的 JEPA 框架第一次具体落在图像上：在抽象表征空间预测，既比 MAE 等像素方法语义更强，又比 iBOT/DINO 等不变性方法省算力、且摆脱手工增广偏置。

2.2 架构与方法

  • 三个 ViT：context-encoder（只编码可见块）、target-encoder（EMA 副本，编码全图产生 patch 级目标）、predictor（窄 ViT，固定嵌入维 384 瓶颈 vs 编码器 1024；深度：ViT-B 用 6、ViT-L/H 用 12、ViT-G 用 16）。
  • 无 [CLS] token，下游用 target-encoder 输出做平均池化得到全局表征。
  • 关键技巧：目标由"掩码 target-encoder 的输出"得到（而非掩码输入像素），保证目标高语义。

多块掩码（Multi-Block Masking）

采样 M=4 个较大的语义 target 块（尺度 0.15–0.2，长宽比 0.75–1.5），以及 1 个信息充分的 context 块（尺度 0.85–1.0），并把与 target 重叠的 context 区域移除（context 仅约占 0.25）。消融显示这是产生语义特征的决定性设计：

其他消融：target 块数 M=1/2/3/4 → 9.0 / 22.0 / 48.5 / 54.2；context 尺度越大越好（0.40→0.85 区间 31.2→54.2）；predictor 深度 12 优于 6（66.9 vs 64.0）；384 维瓶颈优于 1024。

2.4 算力

这是全系列唯一明确给出 GPU-小时的一篇：ViT-H/14 在 ImageNet 上用 16 块 A100、不到 72 小时（即 <1200 GPU-小时）训练完成——比 ViT-S/16 的 iBOT 快 >2.5×，比 ViT-H/14 的 MAE 高效 >10×。优化器 AdamW，batch 2048，LR 1e-4→1e-3（前 15 epoch 预热）再 cosine 衰减到 1e-6，weight decay 0.04→0.4，EMA momentum 0.996→1.0。预训练数据 IN1K（~128 万图），scaling 用 IN22K（ViT-H/14 训练等效 900 个 IN1K epoch）。

3.1 动机

核心问题：仅凭"特征预测"这一个目标（不用预训练编码器、不用文本、不用负样本、不用像素重建），能否从视频学到通用表征？V-JEPA 是 I-JEPA 在视频模态上的直接延伸。

3.2 架构与方法

  • Tubelet 化：16 帧视频经 3D 卷积（tubelet 2×16×16）变成 token 序列（16×224×224 → 8×14×14 = 1568 token），加 3D sin-cos 绝对位置编码。
  • 损失 L2 → L1：沿用 JEPA 但改用 L1 回归，训练更稳定。
  • 注意力探测（Attentive Probe）：冻结评测时用可学习交叉注意力池化（而非平均/线性池化）。这对 JEPA 特征至关重要：在 K400 上 +17.3、SSv2 上 +16.1。

3D 多块掩码（~90% 掩码率）

每个 clip 用两类掩码的并集：short-range（8 个块 × 15%）+ long-range（2 个块 × 70%），每个空间块沿整个时间维延展（时间掩码 100%），平均掩码率约 90%；每 clip 采 2 个掩码。26 组掩码消融表明：高空间+高时间掩码率、多个小块、每样本多掩码，是学到语义的关键（random-tube[0.9] 仅 51.5 vs multi-block 72.9，K400）。

4.1 动机

目标升级为：从"被动观看"的互联网视频中学出能理解、预测、规划物理世界的世界模型，同时尽量少用昂贵稀缺的交互数据。相对 V-JEPA，本文同时沿四个轴 scaling，并新增动作条件世界模型与 LLM 对齐。

4.2 方法：同一内核 + 四轴 scaling + 两阶段

阶段一：V-JEPA 2 编码器

  • 仍是隐空间 L1 掩码去噪，但绝对 sin-cos 位置 → 3D-RoPE 相对位置编码（稳定大模型）。
  • 编码器放大到 ViT-g 1B（ViT-L 300M / ViT-H 600M），predictor 固定为 ViT-s 22M。
  • LR 调度改为 warmup-constant-decay（cooldown），固定 teacher EMA=0.99925。
  • 渐进分辨率训练：warmup/常数阶段用 16 帧/256px，仅在末段 12K 步 cooldown 升到 64 帧/384px → 省 8.4× GPU 时间（否则全程 64×384 的 ViT-g 约需 60 GPU-年）。

阶段二：V-JEPA 2-AC（动作条件）

  • 在冻结的 ViT-g 上加 ~300M 预测器（24 层），用块因果注意力 + 3D-RoPE，依据"动作 + 末端执行器状态"自回归预测下一帧隐表征。
  • 损失 = teacher-forcing L1（T=15）+ 2 步 rollout L1（抑制自回归误差累积）。
  • 仅用 <62 小时无标注 Droid 机器人视频后训练。
  • 规划：在想象的隐状态上做 CEM / MPC 能量最小化——无奖励、无任务特定训练、无环境数据。

5.1 动机：诊断 V-JEPA 2 的"稠密短板"

核心诊断：V-JEPA 2 全局理解很强，但稠密特征图嘈杂、碎片化（分割/深度/跟踪差）。原因是——损失只加在被掩码 token 上，编码器就没有动力把局部信息编进可见（context）token，于是这些可见 token 退化成"register 式全局聚合器"。

5.2 方法：三处结构性改动

① 稠密预测损失

把 L1 预测损失加到所有 token（含可见 token）：L_dense = L_predict + L_context。可见 token 损失按到最近掩码块的距离加权 λ_i = λ/√d_min（λ：视频 0.5 / 图像 0.7），并在 epoch 50–100 渐进预热。

单加 context loss：ADE20K 24.6→33.8，但 SSv2 77.0→62.5（全局退化）。

② 深层自监督 (DSS)

多级 predictor（24 层 vs V-JEPA 2 的 12 层）在 4 个中间编码器层（ViT-G 的 [12,24,36,48]）输出预测；把 3 个中间层输出与最终层拼接、MLP 融合。

恢复并提升全局能力（SSv2 回到 72，IN1K 80.8），还免去多尺度评测。

③ 多模态 tokenizer

视频用 3D 卷积 16×16×2、图像用 2D 卷积 16×16，加可学习模态 token。原生处理图像，不再把图像复制成 16 帧伪视频。

ADE20K 40.8→41.4，识别保持稳定。

外加数据 scaling（VisionMix163M：ImageNet-1M → LVD-142M 图像；视频重加权 SSv2 0.056→0.170、YT-1B 0.188→0.720）与模型 scaling 到 ViT-G(2B)，再蒸馏到 ViT-L/B。

5★.2 方法：它到底怎么发挥作用？

载体是 MAGI-1（SOTA 自回归视频生成器，逐块去噪、因果时序建模，支持 I2V / V2V）。核心思路是：把 MAGI-1 生成的"下一块"，与 V-JEPA 2 对"下一块该长什么样"的预测做对比。

图 3 · V-JEPA 2 作为物理奖励的工作机制。同一段过去 x<t 兵分两路：世界模型预测"应该发生什么"，生成器给出"实际生成了什么"，两者在 V-JEPA 2 表征空间比对得到 surprise 分数，再反过来引导/筛选生成。

① surprise / 符合度 评分（Eq 1）

S( xt | x<t, txt ) = sim(  Pφ( Eθ(x<t) )  ,  Eθ( Gγ(x<t, txt) )  )
# P_φ,E_θ：冻结的 V-JEPA 2 预测器/编码器 · G_γ：MAGI-1 生成器 · sim：余弦相似度

逐项拆解（这就是"怎么发挥作用"）：

  • Eθ(x<t)：V-JEPA 2 把真实的过去块编码进表征空间。
  • Pφ(·)：predictor 据此预测下一块的表征——即"按它学到的直觉物理，接下来合理的画面在表征空间长什么样"。
  • Eθ(Gγ(x<t,txt))：MAGI-1 生成一个候选下一块，再用同一个 E_θ 编码到同一表征空间。
  • sim(·,·)：两者余弦相似度。高 = 生成结果吻合世界模型的物理预期（不"surprise"、物理可信）；低 = 失配、违反物理。

② 两种用法：采样引导 + Best-of-N

(a) 采样引导——在 MAGI-1 原本的扩散得分函数（含无条件项、过去块条件项、文本条件项）后，追加一个由 V-JEPA 2 导出的引导项，把每一步去噪都推向世界模型认为更合理的方向：

∇xt log ps(xt|x<t,txt) = (1−ω<t)∇xtlog p(xt) + (ω<t−ωtxt)∇xtlog p(xt|x<t) + ωtxt∇xtlog p(xt|x<t,txt) − ωs∇xtS(xt|x<t,txt)
# 粉色项为新增的 V-JEPA 2 物理引导，ω_s 控制强度

(b) Best-of-N 筛选——按上式为每个条件采 16 个视频样本，再挑出平均 surprise 最低（与世界模型预测最吻合）的那一个作为最终输出。

5★.4 进阶：从"一个奖励"升格为框架 —— WMReward 与推理时物理对齐

完整版（arXiv 2601.10553）把上述做法提炼成通用框架 WMReward，并给出一个更锋利的论点：

视频生成的物理硬伤，不只是"预训练学得不够"，也是"推理策略不够好"。于是把"改进物理"显式建模为一个推理时对齐（inference-time alignment）问题——在不重训生成器的前提下，用测试时算力去搜索/引导，从一个"向物理可信倾斜"的分布中采样。而且这个框架不绑定 V-JEPA 2：任何具备强物理先验的隐空间世界模型，原则上都能当奖励。

① 倾斜分布（tilted distribution）

p*(x) ∝ w(x) · p(x)
# p(x)：预训练视频模型分布 · w(x)>0：由物理奖励 r(x) 构造的权重 · 目标：从更物理可信的 p* 采样

② 滑动窗口 surprise 奖励（Eq 4–6）——比短报告更精确的形式

在生成视频上滑动长度 C+M 的窗口（C=上下文帧、M=预测视界）：仅用上下文预测未来表征 ẑk，再编码整段窗口得到 zk，只比对未来段，窗口平均的 1−cos 即 surprise（越低越物理连贯）：

ẑk = Pφ( Δm, Eθ(xk−C+1:k) ) # 仅凭上下文预测的未来表征 (Eq 4)
zk = Eθ( xk−C+1:k+M ) # 编码完整窗口得到的表征 (Eq 5)
surprise(x) = (1/|K|) Σk∈K ( 1 − cos( ẑkfut, zkfut ) ) # 越低 = 生成越吻合世界模型预测 (Eq 6)

③ 三种采样方案：怎么从 p* 里采样

方案权重 w(x)类型要点
Guidance (∇)exp(λr(x))梯度式经 Tweedie 公式把 λ∇r(x0|t) 加进扩散 score，每步去噪都被引导；需对 V-JEPA 反传
Best-of-N (BoN)[F(r(x))]N−1免梯度采 N 个粒子，选 r 最高（surprise 最低）者；N=16
∇ + BoN（最佳）exp(λr)·[Fλ(r)]N−1组合引导生成 N 个再挑最优：无需大 λ 即可强倾斜，并过滤近似误差，scaling 最好

④ 测试时算力 scaling

粒子数 N↑ → PhysicsIQ 稳定↑：N≤4 时提升最猛，之后持续小升且方差变小；∇+BoN 的 scaling 曲线比纯 BoN 更陡。把更多算力投到"更多粒子 + 引导"，会把分数分布整体推向高分区——这正是"用推理时计算换物理可信度"的体现。它也胜过 SMC、SVDD 等其他从倾斜分布采样的方法（因为 V-JEPA surprise 可微，其逐步梯度比多次 rollout 的重要性估计更准）。

5★★★.1 洞察：把"物理可信"当约束，而非又一个配权重的奖励项

视频生成模型学的是 2D 像素统计、而非 3D 物理规律，常出现穿模、瞬移、结构畸变。已有 RLHF 对齐难点：人工标注细粒度物理不可扩展；URDF/ControlNet 显式条件不泛化到开放世界；手工把"美学 + 物理"线性配权重会产生梯度冲突。PE-GRPO 的两点洞察：① 用冻结的隐空间世界模型（V-JEPA 2）当免标注、可扩展的物理奖励；② 把物理可行性建模成约束（CMDP）而非固定权重项，并按去噪阶段做信用分配。

5★★★.2 方法：PEARL 三通道奖励 + CMDP 约束 + 相位路由

物理（V-JEPA 2，冻结）

对生成视频采时空掩码，用冻结 V-JEPA 2 预测被掩码的隐 token，与编码目标比 Smooth-L1。预测误差越小 = 越物理一致。

任务（Qwen2-VL-2B，冻结）

不解码硬 Yes/No，而是读 Yes/No token 的 logits → 温度 softmax → R_task = 2·p_Yes − 1 ∈ [−1,1]，得到密集的语义遵循奖励。

时序连续（SSIM，无模型）

相邻帧结构相似度均值 R_smooth = mean SSIM(I_t, I_{t+1})，抑制闪烁/抖动。

# 物理奖励 = 负的「掩码隐 token 预测误差」（Eq 3），由冻结 V-JEPA 2 给出
Rphys = − (1/|M|) Σp∈M (1/d) Σj SmoothL1( ẑp,j , zp,j )
  • CMDP 约束（Auto-PEARL）：不手工配权重，而是 max E[R_task + R_smooth] s.t. E[R_phys] ≥ τ，用拉格朗日乘子 λ 的对偶更新求解；阈值 τ 自动标定（warm-up 期记录 0.8 分位数、取中位数），避免"美学 vs 物理"的梯度冲突。
  • 相位感知路由（sigmoid）：把 macro（任务 + λ·物理）优势分配给早期高噪声步（定场景布局/交互结构），把 micro（SSIM）分配给晚期干净步（去闪烁/细节精修）。
  • 优化：把随机去噪看作 SDE/MDP 策略执行（缓存 log-prob 复用 rollout），用 critic-free 裁剪式 GRPO + KL 拉回参考策略。

10 · 贯穿全系列的开放问题

① 数据多样性天花板

V-JEPA 起就推测公开视频缺乏图像语料的互联网级多样性，故静态图任务逊于 DINOv2/OpenCLIP。即便 scaling 到 22M 视频 + LVD-142M，V-JEPA 2.1 在分割上仍落后 DINOv3-7B（ADE20K 47.9 vs 55.9），归因于训练集"高度杂乱场景"偏少。

② 规模上限

V-JEPA 2 直言只 scaling 到"适度的" 1B（同期他工作已到 20B），>1B 的可持续 scaling 配方仍是未来工作；2.1 到了 2B 却不报算力，性价比无从评估。

③ 稠密 vs 全局的权衡

2.1 的 context loss 单用会拉低全局（SSv2 77.0→62.5），靠深层自监督才挽回——局部 grounding 与全局聚合之间的平衡靠超参（λ、距离加权、预热）调出来，并非已闭合的问题。

④ 规划 / 世界模型的边界

2-AC 对相机位置敏感（需从未标定单目 RGB 推断动作坐标轴，旋转误差随角度近线性增长）；长程规划受自回归误差累积 + 搜索空间指数爆炸限制，取放需中间子目标图；预测仅 ~16 秒；目标是图像而非语言。

⑤ 预判与细时序 QA 缺口

2 未完全攻克 EK100、只聚焦 1 秒预判（更长会骤降）；2.1 反而在 TemporalBench/TOMATO（细运动短时 QA）与 Diving-48/K400 上略退——稠密目标似乎小幅牺牲了细时序/外观锐度。

⑥ 复现与算力透明度

V-JEPA 与 V-JEPA 2.1 均不报告总 GPU-小时/墙钟（前者给样本数，后者啥也不给）。开篇 I-JEPA 那条 <1200 GPU-h 的"高效"叙事在 2B 尺度是否仍成立，无法从论文直接验证。
